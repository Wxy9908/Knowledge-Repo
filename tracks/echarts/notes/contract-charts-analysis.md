# 合同业务看板 — 图表选型分析

> tags: echarts, contract, visualization, product | updated: 2026-07-01 | mastery: learning

## 是什么 / 解决什么问题

本文档定义 Showcase **Tab 3「合同业务看板」** 的 7 张图表：业务含义、对应主流程阶段、ECharts 类型、mock 字段、双受众汇报话术，以及「基础 Tab 简单 / 业务 Tab 复杂」的整体设计原则。

前置阅读：[`contract-business-overview.md`](./contract-business-overview.md)

## 已拍板决策

| 项 | 决定 |
|----|------|
| 图表数量 | **7 图**（合同业务看板 Tab 3） |
| 类型侧重 | 服务 35%、框架 25%（平台搭建期叙事） |
| 框架额度 | **TOP3 横向条形图**（非单 gauge） |
| 受众 | 领导（业务价值）+ 技术（可落地性） |
| 部门 | 虚构 6 个部门 |
| Tab 2 vs Tab 3 | **基础图表从简；业务看板做复杂**，贴合场景并展示 ECharts 能力 |

## Demo 整体设计原则

### Tab 2 · 基础图表（从简）

**目标**：证明「这几种图都会画」，降低认知负担，不与业务耦合。

| 维度 | 策略 |
|------|------|
| 数据 | 通用 mock（与合同无关的月份、品类、分数） |
| 布局 | 2×2 网格，四图等大 |
| option | 最小配置：axis + series + tooltip + legend |
| 交互 | 仅 hover tooltip，无 drill-down、无联动 |
| 样式 | 统一色板，无渐变 / 无双 Y 轴 |

**包含**：折线图、柱状图、饼图、散点图各一。

### Tab 3 · 合同业务看板（做复杂）

**目标**：模拟「合同平台首页驾驶舱」，贴合搭建期（服务 + 框架为主），并集中展示 ECharts 能力。

| 维度 | 策略 |
|------|------|
| 数据 | 统一 `contractData.js`，字段贴近真实合同 |
| 布局 | KPI 条 + 多行不等分网格（上趋势/结构，中运营/专题，下漏斗） |
| option | 丰富配置（见下方「能力展示清单」） |
| 交互 | tooltip 格式化（金额千分位、百分比）、legend 筛选、emphasis |
| 样式 | 面积渐变、markLine 预警、堆叠、富文本标题 |

**ECharts 能力展示清单（业务 Tab 尽量覆盖）**：

- `tooltip.formatter` 自定义（金额、占比）
- `legend` + `selectedMode`
- `series.areaStyle` 面积渐变
- `markLine` 预警线（到期阈值）
- `funnel` 漏斗图
- 横向 `bar` + 双系列（额度已用 / 总额度）或 `label` 百分比
- 环形 `pie` + `emphasis`
- 多图同页 `resize` / `dispose` 生命周期（composable）

## 合同业务主流程（速查）

与 [`contract-business-overview.md`](./contract-business-overview.md) 一致，后文用序号指代：

| 序号 | 阶段 | 一句话 |
|------|------|--------|
| ① | 售前 / 商机 | 有没有单、值不值得跟 |
| ② | 起草与协商 | 写条款、定类型与金额 |
| ③ | 审批与签署 | 走流程、盖章生效 |
| ④ | 履约执行 | 付款、交付、开票、额度扣减 |
| ⑤ | 变更 / 续约 / 终止 | 补充协议、调价、续约 |
| ⑥ | 到期与风险管控 | 快到期了、要不要续 |
| ⑦ | 管理复盘 / 经营分析 | 领导看大盘、看结构、看部门 |

> 一张图可能主要服务某一个阶段，也可能汇总多个阶段的结果（标注为「横切面」）。

|---|----------|----------------|----------|--------------|--------|

| 5 | 框架合同额度 TOP3 | **④ 履约执行**（框架额度随子合同执行而消耗） | 使用率最高的 3 个框架协议，预警额度见底 | 横向条形图（已用 vs 总额度） | **高** |

| 7 | 审批流程漏斗 | **③ 审批与签署** | 起草 → 待审 → 履行 → 完结 的转化与流失 | 漏斗图 funnel | **高** |

### 7 图 × 主流程覆盖示意

```text
① 商机    ② 起草    ③ 审批    ④ 履约    ⑤ 变更    ⑥ 到期    ⑦ 复盘
  ·         ·         █         █         ·         █         █████
                      ↑ ⑦漏斗    ↑ ⑤TOP3              ↑ ⑥到期
                      ↑ ③状态(偏)
```

- **⑦ 复盘类（3 张）**：图 1 趋势、图 2 类型、图 4 部门 — 领导最常看的「大盘」
- **③④ 流程与执行（2 张）**：图 3 状态、图 7 漏斗 — 平台搭建期看「流程通不通」
- **④⑥ 专题（2 张）**：图 5 框架额度、图 6 服务到期 — 贴合当前服务+框架侧重

## 各图详细说明

### 图 1 · 月度签约金额趋势

| 项 | 内容 |
|----|------|
| 主流程阶段 | **⑦ 管理复盘 / 经营分析** |
| 业务问题 | 今年/近一年签约金额是升是降？ |
| 图表类型 | `line` + `areaStyle` 渐变 |
| 数据字段 | `signDate` → 按月聚合 `amount` 求和 |
| Mock 要点 | 12 个月，单位「万元」，有轻微波动即可 |

**领导话术**：「这是合同首页最常看的曲线，一眼看出签约节奏。」

**技术话术**：「按 `signDate` 聚合，后续接 API 只需替换数据源；`areaStyle` 提升可读性。」

---

### 图 2 · 合同类型占比

| 项 | 内容 |
|----|------|
| 主流程阶段 | **⑦ 管理复盘**（类型在 **② 起草与协商** 阶段确定） |
| 业务问题 | 我们签的主要是哪类合同？ |
| 图表类型 | 环形 `pie`，`radius: ['45%', '70%']` |
| 数据字段 | `type` 分组计数 |
| Mock 占比 | 服务 35%、框架 25%、采购 20%、销售 12%、补充 8% |

**领导话术**：「平台还在搭建，服务和框架占六成，符合技术交付与框架协议为主的现状。」

**技术话术**：「`emphasis` 高亮扇区；二期可按 `type` 筛选联动其他图。」

---

### 图 3 · 合同状态分布

| 项 | 内容 |
|----|------|
| 主流程阶段 | **横切面** — 合同在 **③ 审批签署** 与 **④ 履约执行** 各状态的实时快照 |
| 业务问题 | 有多少卡在审批？多少在履行？ |
| 图表类型 | `bar`（可选堆叠：正常 vs 异常） |
| 数据字段 | `status` 分组计数 |
| 状态 | 起草 / 待审批 / 履行中 / 已完结 / 已终止 |

**领导话术**：「状态图看流程健不健康，待审批堆积就要加人手或优化流程。」

**技术话术**：「状态枚举与后端字典一致即可；堆叠系列可扩展异常合同。」

---

### 图 4 · 部门签约额对比

| 项 | 内容 |
|----|------|
| 主流程阶段 | **⑦ 管理复盘** |
| 业务问题 | 哪个部门贡献的签约额最多？ |
| 图表类型 | 横向 `bar`，`yAxis.category` |
| 数据字段 | `dept` + `amount` 求和 |
| 虚拟部门 | 华东销售部、华北销售部、采购中心、技术服务部、法务合规部、运营管理部 |

**领导话术**：「部门对比是组织绩效视角，和技术型团队的虚拟部门命名不冲突。」

**技术话术**：「横向柱适合部门名较长的场景；`label` 显示金额。」

---

### 图 5 · 框架合同额度 TOP3

| 项 | 内容 |
|----|------|
| 主流程阶段 | **④ 履约执行** — 框架协议在生效后随子合同/订单扣减额度 |
| 业务问题 | 哪些框架协议快用完了？ |
| 图表类型 | 横向 `bar`，双系列：`已用额度` / `总额度`，或单系列显示使用率% |
| 数据字段 | `frameworkId`、`frameworkName`、`quotaTotal`、`quotaUsed`（仅 `type=framework`） |
| Mock TOP3 示例 | 「2026 技术服务框架」92%、「设备采购框架」78%、「运维外包框架」65% |

**领导话术**：「框架合同是主从架构，这张图看额度消耗，避免超签。」

**技术话术**：「双系列横向条 + `label` 格式化百分比；数据来源框架主表 + 子合同汇总。」

**ECharts 亮点**：双系列对比、`label.formatter`、`color` 按使用率分段（>90% 警示色）

---

### 图 6 · 服务合同到期分布（未来 90 天）

| 项 | 内容 |
|----|------|
| 主流程阶段 | **⑥ 到期与风险管控** |
| 业务问题 | 接下来哪些月份有一批服务合同要到期？ |
| 图表类型 | `bar` + `markLine`（y = 预警阈值，如每月 ≥8 份） |
| 数据字段 | `expireDate`（`type=service`）→ 按周或按月分桶计数 |
| Mock 要点 | 3 个月分桶，其中一个月略高于预警线 |

**领导话术**：「服务合同到期要续签或验收，提前三个月布局。」

**技术话术**：``markLine.data: [{ yAxis: 8, name: '预警线' }]``；后续可跳转到期清单页。」

**ECharts 亮点**：`markLine`、`itemStyle` 超阈值变色

---

### 图 7 · 审批流程漏斗

| 项 | 内容 |
|----|------|
| 主流程阶段 | **③ 审批与签署**（终点「履行中」衔接 **④ 履约**） |
| 业务问题 | 从起草到完结，哪一步流失最多？ |
| 图表类型 | `funnel` |
| 漏斗层级 | 起草 → 待审批 → 履行中 → 已完结（数量递减） |
| 数据字段 | 按 `status` 映射到漏斗阶段计数 |

**领导话术**：「平台在建，漏斗帮我们看到审批卡点，优化流程有依据。」

**技术话术**：``series.type: 'funnel'``，`sort: 'descending'`，`label.formatter` 显示数量与占比。」

**ECharts 亮点**：漏斗图、转化占比 label

## 汇报叙事顺序（领导 + 技术）

```text
1. KPI 条（30 秒）— 平台规模感
2. 图 ①② — 趋势 + 结构（⑦ 复盘）
3. 图 ③⑦ — 流程是否顺畅（③ 审批）
4. 图 ⑤⑥ — 框架 + 服务专题（④⑥）
5. 图 ④ — 部门贡献（⑦）
6. 切 Tab 1 / Tab 2 — 技术选型与基础能力（各 1 分钟）
```

## 与 Tab 2 的边界

| 能力 | Tab 2 基础 | Tab 3 业务 |
|------|------------|------------|
| 散点图 | ✅ 通用数据 | ❌ 不做（避免重复） |
| 漏斗图 | ❌ | ✅ 图 7 |
| markLine | ❌ | ✅ 图 6 |
| 双系列横向条 | ❌ | ✅ 图 5 |
| 面积渐变 | 可选极简 | ✅ 图 1 |
| 合同 mock | ❌ | ✅ 全部 |

## 复盘

- 能否讲清楚：每张图对应主流程哪一阶段、解决什么业务问题？ → **能**（7 图 + KPI 已落地 Tab 3）
- 何时使用：开发 `ContractDashboardPanel.vue`、编写 mock、汇报排练前
- 沉淀去向：`options/contract/`、`mock/contractAggregates.ts`、`ContractDashboardPanel.vue`
