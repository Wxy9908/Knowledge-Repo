# 合同业务概览

> tags: echarts, contract, visualization, product | updated: 2026-07-01 | mastery: learning

## 是什么 / 解决什么问题

本文档从**产品视角**梳理合同业务的典型工作流、参与角色、关键数据维度，以及可视化（ECharts）可能落地的场景。

用途：

- 为 ECharts 演示 Demo 中的「合同业务看板」提供业务语境
- 作为 `contract-charts-analysis.md`（图表选型）的前置输入
- 汇报时向产品 / 业务 / 技术统一话术

## 合同业务主流程

合同系统可理解为一条 **「从意向到履约再到归档」** 的业务链，通常横跨销售、法务、财务、项目管理等角色。

```text
线索/商机 → 起草协商 → 审批签署 → 履约执行 → 变更/续约 → 到期收尾 → 归档分析
   ↑                                                                              ↓
   └──────────────────────── 数据回流、复盘、风控 ────────────────────────────────┘
```

### 1. 售前 / 商机阶段

| 维度 | 说明 |
|------|------|
| 在做什么 | 识别客户需求、评估是否签约、初步报价 |
| 关键数据 | 预计金额、预计签约时间、客户行业、赢单概率 |
| 可视化需求 | **弱** — 多为 CRM 看板；合同系统内偶见「预计签约 pipeline」 |

### 2. 起草与协商

| 维度 | 说明 |
|------|------|
| 在做什么 | 选模板、填条款、多轮修改、版本对比 |
| 关键数据 | 合同类型、甲乙方、金额、付款方式、交付周期 |
| 可视化需求 | **弱** — 以文档与流程为主；运营侧可能分析「协商轮次 / 停留时长」 |

### 3. 审批与签署

| 维度 | 说明 |
|------|------|
| 在做什么 | 部门审批 → 法务审核 → 领导签批 → 电子签 / 纸质签 |
| 关键数据 | 审批状态、当前节点、停留天数、驳回原因 |
| 可视化需求 | **中** — 各状态数量、审批耗时、瓶颈节点、超期未签预警 |

### 4. 履约执行

| 维度 | 说明 |
|------|------|
| 在做什么 | 按合同付款、交付、开票、验收 |
| 关键数据 | 计划付款日、实际付款、开票金额、里程碑完成情况 |
| 可视化需求 | **强** — 付款计划 vs 实际、履约进度、应收 / 应付汇总 |

### 5. 变更、续约、终止

| 维度 | 说明 |
|------|------|
| 在做什么 | 补充协议、金额调整、续约谈判、提前终止 |
| 关键数据 | 变更次数、变更后金额、续约率 |
| 可视化需求 | **中** — 变更频次趋势、续约率、流失原因分布 |

### 6. 到期与风险管控

| 维度 | 说明 |
|------|------|
| 在做什么 | 到期提醒、续签跟进、违约处理 |
| 关键数据 | 到期日、剩余天数、是否已续签 |
| 可视化需求 | **强** — 未来 30/60/90 天到期数量、大额即将到期预警 |

### 7. 管理复盘 / 经营分析

| 维度 | 说明 |
|------|------|
| 在做什么 | 看业绩、看结构、看部门贡献、支撑管理决策 |
| 关键数据 | 签约金额、签约数量、类型、部门、区域、时间 |
| 可视化需求 | **最强** — 合同系统首页 / 驾驶舱的核心场景 |

## 参与角色与关注点

| 角色 | 典型问题 | 图表偏好 |
|------|----------|----------|
| 业务负责人 / 领导 | 今年签了多少？趋势怎样？哪个部门贡献大？ | 趋势折线、部门对比柱、总览 KPI |
| 销售 / 商务 | pipeline 怎样？哪类合同最好卖？ | 类型占比饼图、个人 / 团队排行 |
| 法务 | 待审多少？卡在哪？平均审多久？ | 状态分布、漏斗、耗时分布 |
| 财务 | 应收应付、开票进度、大额合同 | 金额趋势、区间分布、TOP N |
| 运营 / 管理员 | 系统用得怎么样？流程是否顺畅？ | 流程漏斗、停留时长、异常率 |

**汇报 Demo 默认叙事**：优先讲「经营分析 + 风险预警」，兼顾领导与产品的主要关注点。

## 可视化可能落地的页面场景

按**页面场景**归纳（比按图表类型更易与产品对齐）：

### A. 首页 / 经营驾驶舱（最典型）

一屏多图，回答「整体怎么样」：

| 业务问题 | 推荐图表类型 | Demo 默认是否覆盖 |
|----------|--------------|-------------------|
| 签约金额随时间变化 | 折线 / 面积图 | ✅ 月度签约趋势 |
| 合同类型结构 | 饼图 / 环形图 | ✅ 类型占比 |
| 各状态合同数量 | 柱状 / 堆叠柱 | ✅ 状态分布 |
| 各部门业绩对比 | 横向柱状图 | ✅ 部门对比 |
| 核心 KPI 数字 | 数字卡片（常搭配图表） | 可选，看板顶部 |

### B. 合同列表页统计条

- 当前筛选结果下的金额合计、数量
- 按状态的小条形图（mini bar）
- 完成率类指标可用 gauge

### C. 单合同详情页（二期）

- 付款计划时间轴
- 变更历史金额变化折线
- 履约里程碑进度

首版 Demo 可不实现，列入扩展场景。

### D. 审批 / 流程监控页

- 审批漏斗：起草 → 部门审 → 法务 → 领导 → 已签
- 各节点平均停留时长（柱状）
- 超期未处理数量趋势

适合法务 / 运营向扩展；半天版可仅写入分析文档。

### E. 风险与到期预警页

- 未来 N 天到期合同数量（柱状 + 预警线）
- 大额即将到期 TOP 列表旁配趋势
- 区域签约分布（若有地域维度，可用地图）

### F. 报表 / 导出预览（二期）

- 用户选时间、部门后先出图再导出
- 需筛选器 + 图表联动，复杂度高，首版不做

## 常见数据维度（Mock 设计参考）

后续 Demo 的 `contractData.js` 建议围绕以下字段虚构数据：

| 维度 | 示例字段 | 常用于 |
|------|----------|--------|
| 时间 | `signDate`、`expireDate`、`month` | 趋势、到期预警 |
| 金额 | `amount`、`paidAmount` | 趋势、排行、区间 |
| 状态 | `draft` / `pending` / `active` / `completed` / `terminated` | 状态分布、漏斗 |
| 类型 | 采购 / 销售 / 服务 / 框架 / 补充协议 | 占比饼图 |
| 组织 | `dept`、`region`、负责人 | 部门对比、地图 |
| 客户 | 甲方、行业 | 客户 TOP、行业结构 |
| 风险 | 是否超期、是否大额 | 预警类图表 |

### 状态枚举参考

| 状态值 | 含义 | 中文展示 |
|--------|------|----------|
| `draft` | 起草中 | 起草 |
| `pending` | 审批中 | 待审批 |
| `active` | 履行中 | 履行中 |
| `completed` | 已完结 | 已完结 |
| `terminated` | 已终止 | 已终止 |

### 类型枚举参考

| 类型值 | 含义 |
|--------|------|
| `sales` | 销售合同 |

## 产品结论

> 合同系统的可视化，核心不是「画图」，而是 **在关键决策节点把数据变成可行动的信息**。
>
> 首版 Demo：**7 图合同看板**覆盖主流程 ③④⑥⑦，并以服务 + 框架专题体现搭建期特点。
