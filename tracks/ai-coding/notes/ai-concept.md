## AI基础概念信息

## LLM - Large Language Model 大（语言）模型

- 基于Transformer架构训练出来的
- Google发明的
- 真正推广开来的是Open AI
  - 22年底的GPT-3.5,可用基本的大模型
  - 23年3月 GPT-4，AI能力天花板达到新的高度
  - 如今 GPT-5.4，业界标杆之一

- 大模型本质是文字接龙游戏，所以会产生“幻觉”
- 大模型本质是庞大的数据函数，里面跑矩阵运算
  - 接收数字，输出也是数字
  - 它的能力是根据训练数据来预测下一个词

## Token - 字符

- **大模型处理数据的最基本单位**
- Token：模型自己学会的一套文本切分规则，每一块是它能处理的最小单位。
- Tokenizer：用户和大模型之间的翻译器，通过编码和解码完成用户和大模型之间信息的传递。
- 将话切分成一个个小片段，每一个片段就是一个Token

## Context - 上下文

- **大模型每次处理任务时所接收到的信息总和**
- 类似大模型的临时记忆体，上下文包含的内容越多，冗余信息就会越多，很多细节把握就不会那么清晰了
- **Context Window - 上下文窗口**
  - 代表了Context能容纳的最大Token数量
  - GPT-5.4:105w
  - Gemini 3.1 Pro：100w
  - Claude Opus 4.6：100w
- RAG技术：让大模型不是从庞大的数据库里面找一条信息，而是类似目录，模糊匹配到相关信息，抽离出来，然后再辨析哪一条最符合用户的要求

## Prompt - 提示词

- **用户或系统当前给大模型下达的具体指令或问题**
- prompt等于你要大模型帮你做的事情，prompt怎么写，直接决定大模型返回结果的质量。
  1. User Prompt - 用户提示词（用户输入）
  2. System prompt - 系统提示词（后台配置的）
- **Prompt Engineering - 提示词工程**
  - 研究更好的表述需求的方式

## Tool - 工具

- **大模型用来感知和影响外部环境的函数**
- 大模型无法感知外部环境
- tool本质上就是函数（也可以理解为api）
- 让大模型可以感知外部环境
  - 比如查询天气，查询线路，查询附近好吃的等等等等
  - 直接人需要借助各类APP才能做到的事情，大模型接入相关它们的工具，就可以一键式的实现这些功能

## MCP - Model Context Protocol 模型上下文协议

- **统一了工具接入格式的标准协议**
- 将工具接入平台的时候，不同大模型有不同的接入规范
  - 类似与你开发一个平台，正常是用JS，要放在小程序上，得用小程序的语言，要放在iOS平台上，得用Object C


## Agent

- **能够自主规划，自主调用工具直至完成用户任务的程序**
- 主流产品
  - Claude Code
  - Codex
  - Gemini CLI

## Agent Skill

- **提前规定好，给Agent看的说明文档**
- 我的理解就是给Agent一个角色，Agent有各种能力，当它知道你的侧重的时候，能给你更精准的答案，也可以理解为给Agent一个人设，按照你的偏好给Agent一些规定。
